Lamination Parameters & Realizable Domain Characterization
Studying the feasible LP space achievable with practical composite layups satisfying aerospace design guidelines

Core Theory — Lamination Parameters

  • Lamination parameters VA,B,D1–4 condense layup stiffness into 12 scalar variables (4 in-plane, 4 coupling, 4 bending)
  • A/B/D stiffness matrices are formulated using CLT and/or LPs combined with material invariants
  • Continuous LP optimisation produces ideal stiffness targets — but not all LP combinations correspond to physically realizable discrete layups
  • Challenge: Not all feasible LP combinations satisfy the design guidelines imposed in practice
Realizable Domain
The subset of LP space achievable with practical layups that satisfy all design guidelines

Design Guidelines Studied

  • Disorientation: Maximum angle change between adjacent plies (≤ 45°)
  • Damage Tolerance (DamTol): ±45° plies must be the outermost plies
  • Balance: Equal number of +θ and −θ plies (net in-plane shear coupling = 0)
  • Contiguity: Maximum number of consecutive plies with the same orientation
  • 10% Rule: Minimum 10% of plies in each of 0°, ±45°, 90° families
  • Symmetry: Symmetric mid-plane (B matrix = 0)
Realizable domain in LP space — progressively constrained by applying all six design guidelines
●Optimisation Engineer
Bi-level Optimisation Framework
First level structural optimisation via LPs coupled with second level stacking sequence retrieval constrained to the realizable domain

First Level — Structural Optimisation

  • Minimise mass / maximise safety factor for CFRP panels
  • Design variables: lamination parameters (continuous)
  • Solver: DLR Lightworks — in-house composite panel FEA
  • Constraints: LP feasibility + structural performance targets
  • Output: optimal LP vector (continuous, may lie outside realizable domain)

Second Level — Stacking Sequence Retrieval

  • Given target LPs from first level, find closest discrete layup
  • Search constrained to realizable domain characterised in thesis
  • Tool: LAYLA — open source stacking sequence retrieval code
  • Discrete ply angles: 15° steps in range [−90°, +90°]
  • Minimise LP distance: ‖V_target − V_discrete‖
  • Output: manufacturable ply sequence satisfying all design guidelines

Research Gap — Why This Matters

When the first level optimisation selects LPs outside the realizable domain, the second level retrieval cannot reproduce the target stiffness. This creates a safety factor prediction error — the optimised design on paper does not match the manufacturable layup in practice. This thesis characterises the realizable domain and constrains the first level to it, directly reducing this mismatch.

Interpretation

For LC1–LC4, the first level optimum already lies within the realizable domain — so the derived constraints are inactive and no mismatch is generated in the first place.

For LC5 (compression dominant), the realizable domain constraint brings the first level optimum fully within reach of the second level — 100% mismatch elimination.

For LC6 (tension dominant, Nx = 500,000 N/mm), the LP space is largest, placing the unconstrained optimum furthest from the realizable domain. The derived constraints reduce this mismatch by 69.3%.

●Programming
Python Automation & Subroutine Development
Built Python framework at DLR automating the full LP characterisation → first level → second level pipeline

Modules Developed

  • LP extractor: convert layup sequences to lamination parameters VA,B,D1–4
  • Domain mapper: vector analysis of LP space per design guideline
  • A/B/D builder: formulate stiffness matrices from LPs + material invariants
  • DLR Lightworks interface: automate first level FEA calls, parse SF results
  • LAYLA interface: invoke second level stacking sequence retrieval, collect discrete layup output
  • Mismatch analyser: compare SF from continuous LP vs discrete layup, compute percentage reduction

Technical Stack

  • Python 3.x — core framework
  • NumPy — matrix algebra (A/B/D, LP calculations)
  • SciPy — constrained optimisation (SLSQP)
  • Matplotlib — LP domain plots, result visualisation
  • DLR Lightworks — composite panel FEA, first level solver (DLR in-house tool)
  • LAYLA — open source stacking sequence retrieval, second level solver
●Structural Analysis
CFRP Panel Analysis & Safety Factor Validation
Structural performance of composite panels under combined loading — comparing continuous LP predictions vs retrieved discrete layup results

Panel Configuration

  • Material: CFRP (Carbon Fibre Reinforced Polymer)
  • Stacking sequences: optimised for benchmark load cases from literature
  • Analysis tool: DLR Lightworks (in-house composite FEA — first-ply failure, buckling)
  • Output metrics: safety factors per load case, LP-to-layup mismatch

Validation Approach

  • Step 1: Optimise with unconstrained LPs (first level) → get optimal SF
  • Step 2: Retrieve discrete layup (second level, LAYLA) → evaluate actual SF
  • Step 3: Quantify SF mismatch (baseline)
  • Step 4: Repeat with realizable-domain-constrained first level LPs
  • Step 5: Compare mismatch → demonstrate reduction
