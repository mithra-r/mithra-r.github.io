●Composites Engineer
Classical Lamination Theory & Lamination Parameters
Characterising the feasible design space of composite layups using lamination parameter (LP) framework

Core Theory — Lamination Parameters

  • Lamination parameters (V1–V4, W1–W4) condense layup stiffness into 12 scalar variables
  • A/B/D stiffness matrix fully described via LPs + invariants (CLT)
  • Continuous LP optimisation produces optimal stiffness targets
  • Challenge: not all LP combinations correspond to physically realizable discrete layups
Realizable Domain
The subset of LP space achievable with practical layup sequences

Practical Layup Constraints Studied

  • Ply angle discretisation: 0°, ±45°, 90° families
  • Symmetry: symmetric mid-plane (B=0 requirement)
  • Balance: equal +θ/−θ plies
  • Contiguity: max. consecutive same-angle plies
  • 10% rule: minimum ply fraction per angle

Why This Matters

When continuous LP optimisation selects LPs outside the realizable domain, the retrieved discrete layup cannot reproduce the target stiffness. This creates a safety factor prediction error between the optimised design and the manufacturable layup. This thesis quantifies and reduces that error.

●Optimisation Engineer
Bi-level Optimisation Framework
Outer structural optimisation coupled with inner layup retrieval constrained to the realizable domain

Outer Level — Structural Optimisation

  • Minimise mass / maximise safety factor for CFRP panels
  • Design variables: lamination parameters (continuous)
  • Constraints: LP feasibility + structural performance
  • Solver: Python (SciPy optimise)

Inner Level — Layup Retrieval

  • Given target LPs, find closest discrete layup
  • Search constrained to realizable domain characterised in thesis
  • Minimise LP distance: ‖V_target − V_discrete‖
  • Output: manufacturable ply sequence

Load Cases Evaluated

  • Uniaxial tension — Nxx loading
  • Uniaxial compression — buckling-critical
  • Pure shear — Nxy loading
  • Coupled shear + compression — worst-case interaction

●Programming
Python Automation & Subroutine Development
Developed Python framework automating the full CLT–LP–optimisation–retrieval pipeline at DLR

Modules Developed

  • CLT solver: compute A/B/D matrices from ply angles + thicknesses
  • LP extractor: convert layup sequences to lamination parameters
  • Domain mapper: enumerate and store realizable LP combinations
  • Optimiser wrapper: interface with SciPy for outer/inner levels
  • DLR Lightworks interface: automate FEA calls, parse results

Technical Stack

  • Python 3.x — core framework
  • NumPy — matrix algebra (CLT, LP calculations)
  • SciPy — constrained optimisation (SLSQP, Nelder-Mead)
  • Matplotlib — result visualisation, domain plots
  • DLR Lightworks — composite panel FEA (internal DLR tool)
●Structural Analysis
CFRP Panel Analysis & Safety Factor Validation
Structural performance of composite panels under combined loading, comparing continuous LP vs discrete layup predictions

Panel Configuration

  • Material: CFRP (Carbon Fibre Reinforced Polymer)
  • Stacking sequences: standard aerospace layup families
  • Analysis tool: DLR Lightworks (in-house composite FEA)
  • Output: first-ply failure, buckling load, safety factors

Validation Approach

  • Step 1: Optimise with continuous LPs → get optimal SF
  • Step 2: Retrieve discrete layup → evaluate actual SF
  • Step 3: Quantify mismatch (baseline method)
  • Step 4: Repeat with realizable-domain-constrained LPs
  • Step 5: Compare mismatch → demonstrate improvement
